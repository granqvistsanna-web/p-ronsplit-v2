<task type="auto">

  <files>src/hooks/queries/queryKeys.ts</files>
  <action>
Create queryKeys.ts implementing the query key factory pattern from TkDodo's blog:

1. Import types from ./types

2. Create queryKeys constant object with:

expenses:
- all: ['expenses'] as const - invalidates ALL expense queries
- lists: () => [...queryKeys.expenses.all, 'list'] as const - invalidates all lists
- list: (filters: ExpenseFilters) => serialize filters and include in key
  - Serialize Date objects to ISO strings for stable cache keys
  - Include groupId, serialized dateRange, and memberIds array

incomes:
- all: ['incomes'] as const
- lists: () => [...queryKeys.incomes.all, 'list'] as const
- list: (filters: IncomeFilters) => same serialization pattern

Key serialization requirement: Date objects must be converted to ISO strings because React Query compares keys by value. Date objects would always be different references.

Example key output for expenses.list({ groupId: 'abc', dateRange: { start: Date, end: Date }, memberIds: ['user1'] }):
['expenses', 'list', { groupId: 'abc', dateRange: { start: '2024-01-01T00:00:00.000Z', end: '2024-01-31T23:59:59.999Z' }, memberIds: ['user1'] }]

Add JSDoc comments explaining:
- How to invalidate at different granularities
- Why Date serialization is necessary
  </action>
