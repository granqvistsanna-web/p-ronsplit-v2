Create currency.ts with the following functions:

1. toOre(kr: number): number
   - Converts kronor to ore (multiply by 100, round to integer)
   - Use Math.round() to handle floating-point imprecision
   - Example: toOre(123.45) => 12345
   - Example: toOre(123.456) => 12346 (rounds)

2. toKronor(ore: number): number
   - Converts ore to kronor (divide by 100)
   - Example: toKronor(12345) => 123.45

3. formatCurrency(ore: number, options?: { showDecimals?: boolean }): string
   - Formats ore amount for display with Swedish locale
   - Default: no decimals (1234500 => "12 345 kr")
   - With showDecimals: true (1234500 => "12 345,00 kr")
   - Use toLocaleString('sv-SE') for proper thousand separators

4. parseInputToOre(input: string): number | null
   - Parses user input string to ore
   - Handle both comma (Swedish) and period (international) as decimal separator
   - Remove whitespace/thousands separators
   - Return null for invalid input
   - Example: parseInputToOre("123,45") => 12345
   - Example: parseInputToOre("abc") => null

Add JSDoc comments with examples for each function.

- All monetary values stored as integers (ore) in database
- JavaScript Number.MAX_SAFE_INTEGER is ~9 quadrillion ore (~90 trillion kr) - well above any realistic amount
- This prevents floating-point errors like 0.1 + 0.2 = 0.30000000000000004
