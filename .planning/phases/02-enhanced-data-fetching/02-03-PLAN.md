  <action>
Refactor useExpenses.tsx to use React Query:

1. Keep existing type exports (Expense, ExpenseSplit, ExpenseRepeat) for backward compatibility

2. Change hook signature:
   - Old: useExpenses(groupId?: string)
   - New: useExpenses(filters: ExpenseFilters)
   Where ExpenseFilters has groupId (required), dateRange (optional), memberIds (optional)

3. Replace useState + useEffect + fetchExpenses with useQuery:
   ```typescript
   const query = useQuery({
     queryKey: queryKeys.expenses.list(filters),
     queryFn: async () => {
       // Build Supabase query with filters
       let query = supabase
         .from('expenses')
         .select('*')
         .eq('group_id', filters.groupId)
         .order('date', { ascending: false });

       // Apply date range filter if provided
       if (filters.dateRange) {
         query = query
           .gte('date', filters.dateRange.start.toISOString().split('T')[0])
           .lte('date', filters.dateRange.end.toISOString().split('T')[0]);
       }

       // Apply member filter if provided
       if (filters.memberIds && filters.memberIds.length > 0) {
         query = query.in('paid_by', filters.memberIds);
       }

       const { data, error } = await query;
       if (error) throw error;

       // Parse splits and normalize data
       return (data || []).map(row => ({
         ...row,
         splits: parseSplits(row.splits),
       })) as Expense[];
     },
     enabled: !!filters.groupId, // Only run when groupId available
     staleTime: 30 * 1000, // 30 seconds
     gcTime: 5 * 60 * 1000, // 5 minutes
   });
   ```

4. Convert addExpense, addExpenses, updateExpense, deleteExpense to useMutation:
   - Each mutation should call queryClient.invalidateQueries({ queryKey: queryKeys.expenses.lists() }) in onSuccess
   - Keep toast notifications as they are
   - Keep the undo functionality for deleteExpense

5. Keep parseSplits helper function (internal, not exported)

6. Return object should match current API for backward compatibility:
   ```typescript
   return {
     expenses: query.data ?? [],
     loading: query.isLoading,
     addExpense: addMutation.mutateAsync,
     addExpenses: addExpensesMutation.mutateAsync,
     updateExpense: updateMutation.mutateAsync,
     deleteExpense: deleteMutation.mutateAsync,
     refetch: query.refetch,
   };
   ```

Important: The date comparison uses .toISOString().split('T')[0] to get YYYY-MM-DD format, which matches the database date column format.

Note: Do NOT apply currency conversion yet - the database migration may not have run. Keep amounts as-is for now. Currency conversion will be enabled after migration is confirmed.
