<task type="auto">
  <name>Task 1: Create budgets table migration</name>
  <files>supabase/migrations/20260128_create_budgets_table.sql</files>
  <action>
Create migration file with the following:

1. CREATE TABLE public.budgets with columns:
   - id UUID PRIMARY KEY DEFAULT gen_random_uuid()
   - group_id UUID NOT NULL REFERENCES public.groups(id) ON DELETE CASCADE
   - category TEXT NOT NULL (will match CategoryId type values: mat, boende, transport, etc.)
   - amount BIGINT NOT NULL (stored in ore, consistent with expenses)
   - enabled BOOLEAN NOT NULL DEFAULT true
   - period TEXT NOT NULL DEFAULT 'monthly'
   - created_at TIMESTAMPTZ NOT NULL DEFAULT now()
   - updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
   - UNIQUE(group_id, category, period) constraint

2. CREATE INDEX idx_budgets_group_id ON public.budgets(group_id)
3. CREATE INDEX idx_budgets_category ON public.budgets(category)

4. Add COMMENT ON COLUMN for documentation (amount in ore, category matching CategoryId, etc.)

5. ALTER TABLE public.budgets ENABLE ROW LEVEL SECURITY

6. Create 4 RLS policies following existing expenses pattern:
   - SELECT: "Users can view budgets for their groups"
   - INSERT: "Users can create budgets for their groups"
   - UPDATE: "Users can update budgets for their groups"
   - DELETE: "Users can delete budgets for their groups"

   Each policy uses:
   EXISTS (
     SELECT 1 FROM public.group_members gm
     WHERE gm.group_id = budgets.group_id
     AND gm.user_id = auth.uid()::text
   )

Reference 05-RESEARCH.md "Complete Migration File" section for exact SQL.
  </action>
