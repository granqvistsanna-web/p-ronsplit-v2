
2. **Budget interface:**
```typescript
export interface Budget {
  id: string;
  group_id: string;
  category: string;
  amount: number; // BIGINT stored in ore
  enabled: boolean;
  period: string;
  created_at: string;
  updated_at: string;
}
```

3. **Hook signature:** `export function useBudgets(filters: BudgetFilters)`

4. **Query:** useQuery with:
   - queryKey: queryKeys.budgets.list(filters)
   - queryFn: fetch from supabase budgets table, filter by group_id, optionally filter by period and enabled
   - enabled: !!filters.groupId && !!user
   - staleTime: STALE_TIME_FREQUENT
   - Order by category ascending

5. **saveBudget mutation:** Uses upsert with onConflict: "group_id,category,period"
   - This handles both create and update for same category
   - Success toast: "Budget sparad!"
   - Error toast: "Kunde inte spara budget"
   - Invalidates queryKeys.budgets.lists()

6. **updateBudget mutation:** Standard update by id
   - Success toast: "Budget uppdaterad!"
   - Error toast: "Kunde inte uppdatera budget"
   - Invalidates queryKeys.budgets.lists()

7. **deleteBudget mutation:** Delete by id
   - Success toast: "Budget borttagen!"
   - Error toast: "Kunde inte ta bort budget"
   - Invalidates queryKeys.budgets.lists()

8. **Return object:**
```typescript
return {
  budgets: query.data ?? [],
  loading: query.isLoading,
  saveBudget: saveMutation.mutateAsync,
  updateBudget: async (budgetId: string, updates: Partial<...>) => {...},
  deleteBudget: deleteMutation.mutateAsync,
  refetch: query.refetch,
};
```

Reference 05-RESEARCH.md "Pattern 3: useBudgets Hook Structure" for complete implementation.

Important notes:
- Amount is in ore (integers) - no conversion needed in hook, UI handles conversion
- Use TablesInsert<'budgets'> and TablesUpdate<'budgets'> from generated types
- Swedish toast messages for consistency with useExpenses
