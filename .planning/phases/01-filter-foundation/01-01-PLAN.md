# Plan 01-01: Create useFilterParams hook and date preset utilities

## Objective
Create URL-based filter state management infrastructure using React Router's useSearchParams and date-fns for date range calculations.

## Tasks

<task id="1">
Create src/lib/datePresets.ts with date-fns calculations for all 7 required presets:
- Export DatePreset type: 'this-month' | 'last-month' | 'last-30-days' | 'ytd' | 'last-3-months' | 'last-6-months' | 'last-12-months'
- Export DATE_PRESETS object with Swedish labels (use proper å/ä/ö characters)
- Export getDateRange(preset: string) function that returns { start: Date, end: Date }
- Use date-fns functions: startOfMonth, endOfMonth, subMonths, subDays, startOfYear, startOfDay, endOfDay
- All dates use startOfDay for start and endOfDay for end to include full days
- Fall back to 'this-month' for unknown presets with console.warn
- Export DEFAULT_DATE_PRESET constant = 'this-month'
</task>

<task id="2">
Create src/hooks/useFilterParams.tsx custom hook for URL filter state management:
- Import useSearchParams from 'react-router-dom'
- Import getDateRange and DEFAULT_DATE_PRESET from datePresets
- Read datePreset from ?range= URL param (default: DEFAULT_DATE_PRESET)
- Read memberIds from ?members= URL param as comma-separated array (default: empty array)
- Compute dateRange using useMemo with getDateRange(datePreset)
- Export setDatePreset function that updates URL using callback pattern: setSearchParams(prev => { prev.set('range', preset); return prev; })
- Delete 'range' param when value equals DEFAULT_DATE_PRESET to keep URLs clean
- Export setMemberIds function that updates URL, deleting 'members' param when array is empty
- Export hasActiveFilters boolean (true when datePreset !== DEFAULT_DATE_PRESET || memberIds.length > 0)
- Export resetFilters function that calls setSearchParams({}) to clear all params
- Return interface: { datePreset, dateRange, setDatePreset, memberIds, setMemberIds, hasActiveFilters, resetFilters }
</task>
