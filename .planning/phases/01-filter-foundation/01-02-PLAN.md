# Plan 01-02: Create MemberFilter multi-select component

## Objective
Build accessible multi-select dropdown for household member filtering using shadcn Command + Popover pattern.

## Tasks

<task id="1">
Create src/components/filters/ directory if it doesn't exist.
</task>

<task id="2">
Create src/components/filters/MemberFilter.tsx component with multi-select functionality:
- Import GroupMember type from '@/lib/types'
- Props interface: { members: GroupMember[], selectedIds: string[], onSelectionChange: (ids: string[]) => void, disabled?: boolean }
- Import shadcn components: Popover, PopoverContent, PopoverTrigger, Command, CommandInput, CommandList, CommandItem, CommandEmpty, Checkbox, Button
- Import icons from lucide-react: Users, ChevronDown
- Manage open state with useState<boolean>(false)
- Trigger button displays:
  - When no selection: "Alla medlemmar" with Users icon
  - When partial selection: "{N} av {total} valda" (Swedish for "N of total selected")
  - ChevronDown icon on right that rotates when open
- PopoverContent width: w-[240px]
- CommandInput placeholder: "Sök medlem..." (Swedish)
- CommandEmpty message: "Ingen medlem hittades"
- CommandList with CommandItem for each member:
  - Show Checkbox (checked if member.user_id is in selectedIds)
  - Show member.name
  - onSelect toggles member: if selected, remove from array; else add to array
- Optional quick actions at top: "Välj alla" (Select all) and "Rensa" (Clear) buttons
- Button min height: h-10 (44px touch target)
- Add aria-label="Filtrera efter medlem" to trigger button
- Handle toggle logic: const handleToggle = (userId) => { const newSelection = selectedIds.includes(userId) ? selectedIds.filter(id => id !== userId) : [...selectedIds, userId]; onSelectionChange(newSelection); }
</task>
